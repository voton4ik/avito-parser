# Avito Parser → Telegram
## ❗Если запускать софт без ротации прокси и с маленькой задержкой, то через 1-2 часа работы будут прилетать блокировки по IP от авито❗
## ❗config_proxy - использовать при работе с прокси, config - использовать при запуске софта локально❗

Парсер объявлений с Avito через headless-браузер (Playwright + Chromium).
Отправляет новые объявления в Telegram-группу.

---

## Структура проекта

```
avito_parser/
├── parser.py          ← основной код парсера
├── config.py          ← ВСЕ настройки (заполните перед запуском)
├── requirements.txt   ← зависимости Python
├── Dockerfile         ← для запуска в Docker
├── docker-compose.yml ← удобный запуск контейнера
├── logs/              ← файлы логов (создаётся автоматически)
└── data/              ← SQLite база (создаётся автоматически)
```

## Шаг 1 — Создать Telegram-бота

1. Открыть **@BotFather** в Telegram
2. Отправить `/newbot`
3. Придумать имя и username боту
4. Скопировать токен вида `7123456789:AAFxxxxxxxxxxxxxxx`
5. Вставить в `config.py` → `TELEGRAM_BOT_TOKEN`

---

## Шаг 2 — Добавить бота в группу и получить Chat ID

1. Создать группу в Telegram (или использовать существующую)
2. Добавить своего бота в группу как **администратора**
   (нужны права на отправку сообщений)
3. Узнать ID группы одним из способов:

   **Способ А** — через @userinfobot:
   - Добавить @userinfobot в группу
   - Он напишет ID группы (отрицательное число, напр. `-1001234567890`)
   - Удалить @userinfobot

4. Вставить ID в `config.py` → `TELEGRAM_CHAT_ID`

---

## Шаг 3 — Настроить URL поиска на Avito

1. Зайти на **avito.ru**
2. Ввести запрос: `оперативная память`
3. Настроить нужные фильтры (город, цена, состояние и т.д.)
4. Выбрать сортировку **«По дате»** (обязательно!)
5. Скопировать итоговый URL из адресной строки
6. Вставить в `config.py` → `AVITO_SEARCH_URL`

Пример URL:
```
https://www.avito.ru/moskva/tovary_dlya_kompyutera/operativnaya_pamyat-ASgBAgICAkSSA8YQkEDiQA?s=104
```
> `s=104` в конце URL = сортировка по дате. Без этого параметра
> парсер будет видеть одни и те же объявления!

## Шаг 4 — Установка и запуск (локально)

### Требования
- Python 3.11+
- pip

### Установка

```bash
cd avito_parser

# Создать виртуальное окружение
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate       # Windows

# Установить зависимости
pip install -r requirements.txt

# Установить браузер Chromium (только один раз)
playwright install chromium
playwright install-deps chromium   # Linux: системные зависимости

# Заполнить config.py своими данными
nano config.py

# Запустить
python parser.py
```

---

## Шаг 5 — Запуск через Docker (для сервера)

Docker автоматически установит все зависимости и Chromium.

```bash
# Заполнить config.py своими данными
nano config.py

# Собрать и запустить контейнер
docker compose up -d

# Следить за логами в реальном времени
docker compose logs -f

# Остановить
docker compose down
```

## Дополнительные настройки (config.py)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `MIN_PRICE` | `0` | Мин. цена (0 = не фильтровать) |
| `MAX_PRICE` | `10000` | Макс. цена (0 = не фильтровать) |
| `CHECK_INTERVAL_SECONDS` | `180` | Интервал проверки в секундах |
| `HEADLESS` | `True` | Скрытый режим браузера |

## Как это работает (антидетект)

Парсер использует несколько уровней защиты от блокировки:

1. **Headless Chromium** — полноценный браузер, а не HTTP-запросы
2. **Случайный User-Agent** — из пула реальных агентов
3. **Скрытие webdriver-флага** — Avito не видит автоматизацию
4. **Случайный viewport** — разные разрешения экрана
5. **Имитация прокрутки** — человекоподобное поведение
6. **Случайные задержки** — нет точного расписания запросов
7. **Русская локаль и московский таймзон** — как обычный пользователь

---

## Возможные проблемы

**Капча / блокировка**
- Увеличьте `CHECK_INTERVAL_SECONDS` до 300–600
- Попробуйте запускать с `HEADLESS = False` для отладки
- Рассмотрите использование прокси

**Объявления не находятся**
- Проверьте, что в URL есть `s=104` (сортировка по дате)
- Запустите с `HEADLESS = False` и посмотрите что загружается
